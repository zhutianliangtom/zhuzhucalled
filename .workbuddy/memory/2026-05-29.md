## App 版本管理功能（官网 + 后端 API + 管理后台 + App 更新检测）

- **后端 server.js**：
  - `DB.versions` 数据模型，`saveData/loadData` 持久化到 `data/versions.json`
  - `POST /admin/version/publish`（admin，multipart，上传 APK）→ 存 `uploads/apk/`，写入版本记录
  - `GET /version/latest`（公开）→ App 检测更新用
  - `GET /version/list`（公开）→ 官网/管理后台版本列表
  - `DELETE /admin/version/:id`（admin）→ 删除版本
  - 静态服务 `express.static('uploads/apk')` 提供 APK 下载
- **官网** `backend/views/download.html`：
  - 独立页面，渐变紫色主题，显示最新版本号、更新日志、功能亮点
  - 自动从 `GET /version/latest` 获取信息，点下载按钮直接下载 APK
  - 路由 `GET /download` 渲染此页面
- **管理后台** `backend/admin.html`：
  - sidebar 新增「版本管理」入口
  - 版本列表表格（版本号/版本码/更新日志/APK 下载链接/强制更新/操作）
  - 发布新版本弹窗（版本号、版本码、更新日志 textarea、APK 文件选择、强制更新勾选）
- **App.vue**：
  - `onLaunch` 调用 `checkUpdate()`，请求 `GET /version/latest`
  - 发现新版本 → `uni.showModal` 提示更新日志，点确定打开浏览器前往 `/download` 页面
  - 强制更新时不允许取消，取消则退出 App
  - `apiBase` 需手动改为后端实际地址（如 `http://183.66.27.20:5000`）
- **注意**：`App.vue` 中 `plus.runtime.versionCode` 需要打包时配置 `manifest.json` 的 `versionCode`，每次发版需递增
