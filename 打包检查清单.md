## 🚀 打包步骤

### Android APK 打包（云打包 - 推荐）

1. **打开 HBuilderX**
2. **菜单 → 发行 → 原生App-云打包**
3. **选择平台**: Android
4. **证书选择**:
   - 使用公共测试证书（快速测试）
   - 或使用自有证书（正式发布）
     - 证书别名: campus_lostfound
     - 证书密码: [设置强密码]
     - 私钥密码: [同上]
5. **等待打包完成**（约 5-10 分钟）
6. **下载 APK 文件**

### Android APK 打包（本地打包）

1. **安装 Android Studio**
2. **HBuilderX → 发行 → 原生App-本地打包 → 生成本地打包App资源**
3. **Android Studio 打开项目**:
   - 文件 → 打开 → 选择 `unpackage/dist/build/app` 目录
4. **配置签名**:
   - Build → Generate Signed Bundle / APK
   - 选择 APK
   - 选择或创建密钥库
5. **构建 APK**:
   - 选择 Build Variants: release
   - 点击 Finish

### iOS IPA 打包

1. **准备 Apple Developer 账号**
2. **创建 App ID**: com.campus.lostfound
3. **创建证书和配置文件**
4. **HBuilderX → 发行 → 原生App-云打包**
5. **选择平台**: iOS
6. **上传证书和配置文件**
7. **等待打包完成**
8. **通过 TestFlight 或直接安装测试**

## ⚠️ 注意事项

### 1. 版本号管理
- 每次发布必须递增 versionCode
- versionName 格式：主版本.次版本.修订号
- 示例：1.1.2 → 1.1.3 或 1.2.0

### 2. 签名证书
- **妥善保管证书文件**（.keystore 或 .jks）
- 丢失证书将无法更新已发布的 App
- 建议备份到多个安全位置

### 3. 网络配置
- 当前使用 HTTP，生产环境建议启用 HTTPS
- `usesCleartextTraffic: true` 仅用于开发测试

### 4. 权限说明
- Android 13+ 需要动态申请通知权限
- 存储权限在 Android 13+ 已细分为媒体类型
- 确保在 App 中正确处理权限请求

### 5. 应用商店要求
- **Google Play**: 需要 targetSdkVersion >= 34
- **App Store**: 需要支持 64 位架构
- 准备好应用截图、描述、隐私政策
