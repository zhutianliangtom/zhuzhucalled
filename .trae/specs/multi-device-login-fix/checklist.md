# Checklist: 多设备登录顶号功能修复

## 验收清单

### 后端修复
- [x] 登录接口添加会话记录：在 `/auth/login` 中调用 `addSession`
- [x] 会话数据结构正确：包含 userId、token、deviceId、loginTime
- [x] hasOtherDevice 检测逻辑正确：基于活跃会话数量判断

### 前端优化
- [x] 顶号确认弹窗正确显示：标题"登录提示"，内容"该账号已在其他设备登录，是否顶号登录？"
- [x] 取消按钮正常工作：清空密码，保持在登录页
- [x] 确认按钮触发强制登录：调用 `api.forceLogin`

### 功能测试
- [ ] 单设备首次登录：正常登录，`hasOtherDevice: false`
- [ ] 第二设备登录：显示顶号确认，`hasOtherDevice: true`
- [ ] 确认顶号后：新设备登录成功，旧设备被下线
- [ ] 取消顶号：旧设备保持登录，新设备不登录
- [ ] 心跳检测：被顶号后自动退出登录

### 错误处理
- [ ] 网络异常：心跳失败不强制退出
- [ ] token 过期：正常返回 401
- [ ] 后端重启：会话丢失，用户需重新登录
