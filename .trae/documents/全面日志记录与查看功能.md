# 全面日志记录与查看功能

## 1. Summary

为项目添加全面的 API 请求日志记录和实时查看功能：
1. 在 server.js 中添加 HTTP 请求/响应中间件，记录所有 API 请求（Android 前端和管理员面板端）
2. 扩展现有日志机制（server_log/ 目录），记录请求方法、路径、IP、用户、状态码、耗时等
3. 新增 `/admin/logs` 路由和 logs.ejs 页面，管理员可查看实时日志（WebSocket 推送）并下载日志文件
4. 在所有管理员 EJS 页面的侧边栏添加"日志查看"菜单项

## 3. Proposed Changes

### 3.1 server.js - 添加 HTTP 请求日志中间件

**位置**：server.js，在现有的 CORS/bodyParser 等中间件之后（约第 929 行之后），在所有 API 路由之前

**内容**：添加一个 express 中间件，记录所有请求和响应

```javascript
// 请求日志中间件 - 记录所有 HTTP 请求
app.use((req, res, next) => {
  const start = Date.now()
  const timestamp = new Date().toISOString().replace('T', ' ').replace('Z', '')
  const ip = req.headers['x-forwarded-for'] || req.socket.remoteAddress || 'unknown'
  const method = req.method
  const url = req.url
  const userAgent = req.headers['user-agent'] || ''
  
  // 获取当前登录用户信息（如果有）
  let userId = 'anonymous'
  let userDetail = 'anonymous'
  const authHeader = req.headers['authorization']
  if (authHeader) {
    try {
      const token = authHeader.split(' ')[1]
      const decoded = jwt.verify(token, JWT_SECRET)
      userId = decoded.id
      // 尝试从 req.user 获取名称（如果 authenticateToken 已执行）
      if (req.user) {
        userDetail = `${req.user.name || ''}(${req.user.studentId || ''})`
      } else {
        userDetail = `user:${decoded.id}`
      }
    } catch (e) {
      // token 无效，保持 anonymous
    }
  }
  
  // 判断是否是管理员请求
  const isAdminRequest = req.url.startsWith('/admin')
  
  // 捕获响应结束事件
  const originalEnd = res.end
  res.end = function(...args) {
    const duration = Date.now() - start
    const status = res.statusCode
    const logLevel = status >= 500 ? 'error' : status >= 400 ? 'warn' : 'info'
    
    const logMsg = `[${timestamp}] ${method} ${url} ${status} ${duration}ms ${ip} ${userDetail}${isAdminRequest ? ' [ADMIN]' : ''}${userAgent.includes('uniapp') ? ' [APP]' : ''}${userAgent.includes('Mozilla') && !userAgent.includes('uniapp') ? ' [WEB]' : ''}`
    
    log(logLevel, logMsg)
    
    originalEnd.apply(res, args)
  }
  
  next()
})
```

**作用**：
- 记录每个请求的时间戳、方法、URL、状态码、响应耗时、客户端 IP
- 自动解析 JWT token 获取用户 ID
- 标记请求来源（ADMIN / APP / WEB）
- 通过现有 `log()` 函数写入缓冲区，支持实时 WebSocket 推送和文件持久化

### 3.2 server.js - 添加日志实时推送的 WebSocket 支持

**位置**：server.js 中 WebSocket connection 处理部分（约第 283 行之后），当管理员连接时

**内容**：将日志缓冲区内容通过 WebSocket 实时推送给管理员

在现有 WebSocket message 处理中，添加新消息类型：

```javascript
ws.on('message', (data) => {
  try {
    const message = JSON.parse(data)
    
    // 新增：管理员请求实时日志
    if (message.type === 'subscribe_logs') {
      ws.isAdminLogSubscriber = true
      // 立即推送当前缓冲区的日志
      const todayLog = getLogFileName()
      if (logBuffer[todayLog] && logBuffer[todayLog].length > 0) {
        ws.send(JSON.stringify({
          type: 'log_lines',
          content: logBuffer[todayLog]
        }))
      }
    }
  } catch (err) { ... }
})
```

在现有日志记录的 `log()` 函数中（约第 152 行），发送新日志行给所有订阅的管理员客户端：

```javascript
// 在 log() 函数中，写入缓冲区后新增：
// 推送新日志行给订阅的管理员 WebSocket 客户端
for (const client of adminClients) {
  if (client.isAdminLogSubscriber && client.readyState === WebSocket.OPEN) {
    client.send(JSON.stringify({
      type: 'log_line',
      line: logLine
    }))

```

### 3.3 server.js - 添加日志查看 API 路由

**位置**：server.js 中其他 `/admin/*` 路由附近

**新增路由 1**：获取日志内容（用于加载历史日志和轮询增量）

```javascript
app.get('/admin/logs', authMiddleware, async (req, res) => {
  const { date = null } = req.query
  
  try {
    const logDate = date || getLogFileName()
    const logPath = path.join(LOG_DIR, logDate)
    
    let content = ''
    let exists = false
    if (fs.existsSync(logPath)) {
      content = fs.readFileSync(logPath, 'utf8')
      exists = true
    }
    
    // 追加当前缓冲区中未刷盘的日志
    if (logBuffer[logDate]) {
      content += logBuffer[logDate]
    }
    
    res.json({
      success: true,
      content: content,
      date: logDate,
      exists: exists
    })
  } catch (err) {
    res.status(500).json({ success: false, message: err.message })
  }
})

// 获取所有可用的日志日期列表
app.get('/admin/logs/dates', authMiddleware, (req, res) => {
  try {
    const files = fs.readdirSync(LOG_DIR).filter(f => f.endsWith('.log')).sort().reverse()
    res.json({ success: true, dates: files })
  } catch (err) {
    res.status(500).json({ success: false, message: err.message })
  }
})
```

**新增路由 2**：下载日志文件

```javascript
app.get('/admin/logs/download/:filename', authMiddleware, (req, res) => {
  const { filename } = req.params
  // 安全校验：防止路径遍历攻击
  if (!/^\d{4}-\d{2}-\d{2}\.log$/.test(filename)) {
    return res.status(400).json({ success: false, message: '无效的文件名' })
  }
  
  const logPath = path.join(LOG_DIR, filename)
  if (!fs.existsSync(logPath)) {
    return res.status(404).json({ success: false, message: '日志文件不存在' })
  }
  
  res.download(logPath, filename)
})
```

## 4. Assumptions & Decisions

1. **复用现有日志机制**：不新建日志系统，复用现有的 `log()` 函数、`logBuffer` 和 `LOG_DIR`，保持代码一致性
2. **日志格式**：统一使用 `[timestamp] [LEVEL] method url status durationms ip user [SOURCE]` 格式
3. **实时推送**：通过 WebSocket 推送新日志行（低延迟），同时提供 HTTP 轮询 fallback（可靠性）
4. **性能考虑**：
   - 日志仅记录到内存缓冲区，定期刷盘，不阻塞请求
   - 前端限制显示最近 5000 行，避免浏览器卡顿
   - WebSocket 推送逐行增量发送，不一次性推送全部内容
5. **安全**：下载接口使用正则校验文件名防止路径遍历攻击
